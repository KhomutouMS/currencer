# Диаграмма вариантов использования Currencer

![ ](https://github.com/KhomutouMS/currencer/blob/master/Documentation/Diagrams/Images/UseCAse.png)

# Глоссарий

| Термин | Определение |
|:--|:--|
| **Пользователь** | Человек, использующий приложение Currencer |
| **Гость** | Пользователь, использующий приложение без авторизации |
| **Зарегистрированный пользователь** | Пользователь, прошедший регистрацию и авторизацию в приложении |
| **Валютная пара** | Пара валют, например EUR/USD, где первая валюта - базовая, вторая - котируемая |
| **Избранное** | Список валютных пар, отслеживаемых пользователем |
| **Курс валюты** | Отношение стоимости одной валюты к другой |
| **Конвертация** | Процесс пересчета суммы из одной валюты в другую |
| **Симуляция обмена** | Расчет потенциальных результатов обмена валют без реального совершения операции |
| **Уведомление** | Оповещение пользователя о значительных изменениях курсов валют |
| **Отчет** | Сводка на основе истории операций и изменений курсов |
| **Кошелек** | Виртуальный контейнер для отслеживания сумм в различных валютах |
| **Внешние API** | Сторонние сервисы, предоставляющие данные о курсах валют |

<a name="use_case"/>

## 2 Варианты использования

<a name="sign_up"/>

### 2.1 Зарегистрироваться

**Описание.** Позволяет пользователю создать учетную запись в приложении.  
**Предусловия.** Пользователь выбрал опцию "Регистрация".  
**Основной поток.**
1. Приложение отображает форму регистрации;
2. Пользователь вводит email и пароль;
3. Пользователь подтверждает регистрацию;
4. Приложение проверяет уникальность email;
5. Приложение создает учетную запись пользователя;
6. Приложение отправляет подтверждение на email;
7. Вариант использования завершается.

**Альтернативный поток А1.** Email уже существует
1. Приложение сообщает о существовании пользователя с таким email;
2. Возврат к п.2 основного потока.

<a name="sign_in"/>

### 2.2 Войти в систему

**Описание.** Позволяет пользователю войти в свою учетную запись.  
**Предусловия.** Пользователь не авторизован в системе.  
**Основной поток.**
1. Приложение отображает форму входа;
2. Пользователь вводит email и пароль;
3. Пользователь подтверждает вход;
4. Приложение проверяет credentials;
5. Приложение предоставляет доступ к персонализированным функциям;
6. Вариант использования завершается.

<a name="sign_out"/>

### 2.3 Выйти из системы

**Описание.** Позволяет пользователю выйти из учетной записи.  
**Предусловия.** Пользователь авторизован в системе.  
**Основной поток.**
1. Пользователь выбирает опцию "Выйти";
2. Приложение завершает сессию пользователя;
3. Приложение перенаправляет на главный экран;
4. Вариант использования завершается.

<a name="view_currency_rates"/>

### 2.4 Просмотр курсов валют

**Описание.** Отображает актуальные курсы валют.  
**Предусловия.** Приложение имеет доступ к интернету.  
**Основной поток.**
1. Пользователь открывает главный экран приложения;
2. Приложение запрашивает данные у внешних API;
3. Приложение отображает текущие курсы валют;
4. Вариант использования завершается.

**Поток ошибки Е1.** Нет подключения к интернету
1. Приложение отображает сообщение об ошибке;
2. Приложение показывает кэшированные данные (если есть);
3. Вариант использования завершается.

<a name="add_to_favorites"/>

### 2.5 Добавить валютную пару в избранное

**Описание.** Добавляет валютную пару в список отслеживаемых.  
**Предусловия.** Пользователь авторизован и просматривает курсы валют.  
**Основной поток.**
1. Пользователь выбирает валютную пару;
2. Пользователь нажимает "Добавить в избранное";
3. Приложение добавляет пару в список избранного пользователя;
4. Приложение начинает отслеживание изменений курса для этой пары;
5. Вариант использования завершается.

<a name="remove_from_favorites"/>

### 2.6 Удалить из избранного

**Описание.** Удаляет валютную пару из списка отслеживаемых.  
**Предусловия.** Пользователь авторизован и имеет избранные пары.  
**Основной поток.**
1. Пользователь открывает список избранного;
2. Пользователь выбирает валютную пару для удаления;
3. Пользователь подтверждает удаление;
4. Приложение удаляет пару из списка избранного;
5. Приложение прекращает отслеживание изменений курса;
6. Вариант использования завершается.

<a name="convert_currency"/>

### 2.7 Конвертировать валюту

**Описание.** Выполняет расчет конвертации суммы между валютами.  
**Предусловия.** Доступны актуальные курсы валют.  
**Основной поток.**
1. Пользователь открывает конвертер валют;
2. Пользователь выбирает исходную и целевую валюты;
3. Пользователь вводит сумму для конвертации;
4. Приложение рассчитывает результат по текущему курсу;
5. Приложение отображает результат конвертации;
6. Приложение сохраняет операцию в истории;
7. Вариант использования завершается.

<a name="simulate_exchange"/>

### 2.8 Симуляция обмена валют

**Описание.** Моделирует процесс обмена валют с учетом комиссий.  
**Предусловия.** Пользователь авторизован.  
**Основной поток.**
1. Пользователь открывает инструмент симуляции;
2. Пользователь выбирает валюты и суммы;
3. Пользователь настраивает параметры обмена;
4. Приложение рассчитывает потенциальный результат;
5. Приложение отображает детализированный отчет симуляции;
6. Вариант использования завершается.

<a name="view_history"/>

### 2.9 Просмотр истории операций

**Описание.** Отображает историю конвертаций и операций пользователя.  
**Предусловия.** Пользователь авторизован и имеет историю операций.  
**Основной поток.**
1. Пользователь открывает раздел истории;
2. Приложение загружает историю операций пользователя;
3. Приложение отображает операции в хронологическом порядке;
4. Пользователь может фильтровать и сортировать операции;
5. Вариант использования завершается.

<a name="generate_reports"/>

### 2.10 Генерация отчетов

**Описание.** Создает отчеты на основе истории операций.  
**Предусловия.** Пользователь авторизован и имеет историю операций.  
**Основной поток.**
1. Пользователь открывает раздел отчетов;
2. Пользователь выбирает тип отчета и период;
3. Приложение генерирует отчет на основе данных;
4. Приложение отображает отчет в удобном формате;
5. Пользователь может экспортировать отчет;
6. Вариант использования завершается.

<a name="configure_notifications"/>

### 2.11 Настройка уведомлений

**Описание.** Настраивает параметры получения уведомлений.  
**Предусловия.** Пользователь авторизован.  
**Основной поток.**
1. Пользователь открывает настройки уведомлений;
2. Пользователь выбирает типы уведомлений;
3. Пользователь устанавливает пороги для оповещений;
4. Приложение сохраняет настройки;
5. Вариант использования завершается.

<a name="receive_notifications"/>

### 2.12 Получение уведомлений

**Описание.** Получение оповещений о значительных изменениях курсов.  
**Предусловия.** Пользователь настроил уведомления для валютных пар.  
**Основной поток.**
1. Приложение отслеживает изменения курсов для избранных пар;
2. При обнаружении значительного изменения приложение генерирует уведомление;
3. Приложение отображает уведомление пользователю;
4. Вариант использования завершается.

Этот комплект документации полностью покрывает функциональность, описанную в вашем SRS, и предоставляет четкое руководство для разработки системы "Currencer".
